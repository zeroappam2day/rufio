# Research Workflow: 100s of Google Results into Database via Ruflo Swarm

## How It Actually Works

Ruflo's swarm agents do NOT have built-in web search or crawl tools. The swarm provides:
- **Orchestration** — parallel task coordination across agents
- **Memory** — vector-embedded storage with semantic search
- **Learning** — pattern recognition that improves over time

Web searching and crawling is done by **Claude Code** (WebSearch, WebFetch tools) or by **Claude Code skills** (/crawl4ai, /browse, /firecrawl). The swarm stores, indexes, and analyzes the results.

The workflow: **Claude Code searches/crawls -> ruflo swarm stores/indexes/analyzes -> SQLite database**.

The OpenRouter model (nvidia/nemotron-3-super-120b-a12b:free) is used as the LLM brain for agent reasoning and analysis, not for web fetching.

## Pre-Flight (1 min)

```powershell
pm2 resurrect                                    # start MCP server
pm2 list                                         # confirm ruflo-mcp = online
npx ruflo@latest swarm init --v3-mode            # init swarm
npx ruflo@latest doctor                          # quick health check
```

## The Workflow

### Phase 1: Search Google (Claude Code does this)

Open Claude Code in the `227_rufio` directory. Paste this prompt:

```
Search Google for "[YOUR TOPIC]". I need at least 50 results.
For each result, extract:
- URL
- Title
- 2-3 sentence summary of what the page covers

Return all results as a numbered list. Do NOT crawl the pages yet.
Just get the search result snippets.
```

Claude Code will use its WebSearch tool. It can do ~10-20 results per search query. Ask it to search with multiple query variations to get 100+.

### Phase 2: Crawl Top Results (Claude Code does this)

```
Now crawl the top 50 URLs from the search results above.
For each URL:
1. Fetch the page content
2. Extract: title, author, date, key findings (3-5 bullet points), full text summary (100 words max)
3. Store each result in ruflo memory by running:

npx ruflo@latest memory store --key "result-NNN" --value "TITLE | URL | FINDINGS" --namespace research

Process in batches of 10. After each batch, confirm how many were stored.
```

Or use the crawl4ai skill for better extraction:

```
/crawl4ai

Crawl these 50 URLs and extract structured data from each page.
Store each in ruflo memory with namespace "research".
```

### Phase 3: Spawn Analysis Agents (ruflo swarm does this)

```powershell
# Spawn agents for analysis
npx ruflo@latest agent spawn -t researcher --provider openrouter --model nvidia/nemotron-3-super-120b-a12b:free
npx ruflo@latest agent spawn -t analyst --provider openrouter --model nvidia/nemotron-3-super-120b-a12b:free
```

Then in Claude Code:

```
Search the ruflo memory database for all results in the "research" namespace.
Run: npx ruflo@latest memory search --query "[YOUR RESEARCH QUESTION]"

Analyze the top results. Identify:
1. Common themes across all sources
2. Contradictions between sources
3. Key data points with citations
4. Gaps in the research
5. Top 10 most relevant findings

Store the analysis summary in ruflo memory:
npx ruflo@latest memory store --key "analysis-summary" --value "YOUR ANALYSIS" --namespace research-analysis
```

### Phase 4: Query and Export

```powershell
# Semantic search — ask questions across all stored results
npx ruflo@latest memory search --query "what do sources say about X"
npx ruflo@latest memory search --query "pricing trends"
npx ruflo@latest memory search --query "contradictions and disagreements"

# List everything
npx ruflo@latest memory list --namespace research

# Stats
npx ruflo@latest memory stats

# Export to SQL file
npx ruflo@latest memory export --format sql --output C:/Users/anujd/Documents/01_AI/227_rufio/research-export.sql
```

The SQLite database is at `.swarm/memory.db` — open it with DB Browser for SQLite or DBeaver.

## Database Schema

```
memory_entries table:
  id          TEXT    — unique ID
  key         TEXT    — your key (result-001, analysis-summary, etc.)
  namespace   TEXT    — "research" or "research-analysis"
  content     TEXT    — the stored text
  embedding   TEXT    — 384-dim vector (auto-generated by ONNX)
  type        TEXT    — entry type
  created_at  INTEGER — timestamp
  metadata    TEXT    — JSON metadata
```

## What the OpenRouter Model Does

The `nvidia/nemotron-3-super-120b-a12b:free` model is available for:
- Spawned agent reasoning (when --provider openrouter is used)
- Complex analysis tasks that need more compute than local qwen3:8b
- The `webResearchAgent` in agentic-flow (delegates to Claude Agent SDK)

It does NOT directly fetch web pages. Claude Code's built-in WebSearch/WebFetch tools handle that.

## Cost

- Total: $0 additional cost

## Tips

1. **Start with 20 results** to validate the pipeline before scaling to 100+.
2. **Use different namespaces** per research topic: `--namespace topic-A`, `--namespace topic-B`.
3. **Semantic search is the payoff.** "Which results discuss X?" returns ranked results by conceptual similarity, not keyword match.
4. **Database persists across sessions.** Data in `.swarm/memory.db` survives restarts.
5. **Run `npx ruflo@latest embeddings warmup`** if first search is slow (>500ms).
6. **Batch crawling in groups of 10** avoids timeouts and lets you monitor progress.
