# Rufio

AI agent orchestration workspace for Windows 11. Coordinates multiple AI agents using local LLMs (Ollama) and cloud fallback (OpenRouter), with persistent vector-searchable memory.

## What This Does

Rufio lets you spin up teams of AI agents that can research, code, review, and analyze — all coordinated through a single CLI. Think of it as a command center where you dispatch AI workers, and they share a collective memory that gets smarter over time.

**Key capabilities:**
- **Multi-agent swarms** — up to 15 agents working in parallel with hierarchical coordination
- **Hybrid memory** — SQLite database with 384-dimensional vector embeddings (HNSW index) for semantic search
- **Local-first LLMs** — Ollama runs models on your GPU (free, private), OpenRouter handles overflow
- **Research pipelines** — search Google, crawl pages, store findings in a searchable database, run semantic queries
- **Self-learning hooks** — the system learns patterns from your usage and improves routing over time

## Setup

```powershell
# 1. Clone and install
git clone https://github.com/zeroappam2day/rufio.git
cd rufio
npm install

# 2. Create your .env from template
copy .env.example .env
# Edit .env — add your OpenRouter API key

# 3. Start MCP server (persistent background process)
pm2 start "npx @claude-flow/cli@latest mcp start" --name ruflo-mcp
pm2 save

# 4. Initialize swarm
npx ruflo@latest swarm init --v3-mode

# 5. Health check
npx ruflo@latest doctor --fix
```

## Everyday Commands

```powershell
# Restore MCP server after reboot
pm2 resurrect

# Spawn agents
npx ruflo@latest agent spawn -t coder
npx ruflo@latest agent spawn -t researcher

# Store and search memory
npx ruflo@latest memory store --key "finding-1" --value "key insight here" --namespace research
npx ruflo@latest memory search --query "what did we find about X"

# Check health
npx ruflo@latest doctor
```

## How to Rollback

If anything breaks after a change:

```powershell
# Undo all uncommitted changes
git restore .
git clean -fd

# Go back to a specific commit
git log --oneline          # find the commit hash
git reset --hard <hash>    # reset to that point

# Nuclear option — re-clone
git clone https://github.com/zeroappam2day/rufio.git rufio-fresh
```

**What's safe to delete without breaking anything:**
- `.swarm/` — regenerated on swarm init
- `.claude-flow/data/` — regenerated on memory init
- `.claude-flow/neural/` — regenerated on hooks pretrain
- `node_modules/` — regenerated by `npm install`
- `state/scratchpad.json` — ephemeral runtime data

**What you must NOT delete:**
- `.env` — your API keys (not in git, only on your machine)
- `.mcp.json` — MCP server wiring (pain to recreate)
- `claude-flow.config.json` — provider and memory config
- `.claude-flow/config.yaml` — runtime topology and memory settings

## Architecture

```
You (Claude Code) --> MCP Server (pm2) --> Ruflo CLI --> Agent Swarm
                                              |
                                    Ollama (local GPU)
                                    OpenRouter (cloud)
                                              |
                                    Hybrid Memory (SQLite + HNSW vectors)
```

- **MCP Server** runs as a persistent background process via pm2. It exposes 310+ tools to Claude Code.
- **Agent Swarm** coordinates up to 15 agents in a hierarchical-mesh topology. A leader agent distributes tasks.
- **Hybrid Memory** stores data in SQLite with ONNX-generated 384-dim vector embeddings. Semantic search finds conceptually related entries, not just keyword matches.
- **Hooks** intercept operations (pre-task, post-task, pre-edit, etc.) and learn patterns for smarter routing.

## Security Notes

- API keys live in `.env` only — never in committed files
- `.mcp.json` uses `${OPENROUTER_API_KEY}` placeholder — resolved at runtime from `.env`
- `claude-flow.config.json` uses `${OPENROUTER_API_KEY}` placeholder — same pattern
- If you see a raw API key in any committed file, rotate it immediately

## Scaling

- **More agents**: increase `maxAgents` in `.claude-flow/config.yaml` (currently 15)
- **Better models**: swap `OLLAMA_MODEL` in `.env` to a larger model (needs more VRAM)
- **More memory**: HNSW index scales to millions of vectors; increase `cacheSize` in config
- **Cloud-heavy**: set OpenRouter as primary provider in `claude-flow.config.json`
- **Multiple projects**: each project gets its own rufio directory with independent memory

## Troubleshooting

| Problem | Fix |
|---------|-----|
| "MCP server not running" | `pm2 resurrect` or `pm2 restart ruflo-mcp` |
| "Memory Database: Not initialized" | `npx ruflo@latest doctor --fix` |
| Agent spawn hangs | Check Ollama is running: `ollama list` |
| HNSW shows "Available: No" | Cosmetic — works during actual operations |
| "embeddings init" command not found | Memory init handles embeddings internally |
| Swarm won't start | `npx ruflo@latest swarm init --v3-mode` |
